## Нагрузочное тестирование (k6)

Нагрузочное тестирование проводилось для оценки производительности и стабильности системы **Единое хранилище данных (ЕХД)**.  
Тесты написаны с использованием инструмента [k6](https://k6.io/).  
Сценарии тестов находятся в директории `test/`, а результаты прогонов (логи, метрики, отчёты) — в директории `results/`.

### Тестовый стенд

- **СУБД (тестовая база)** развёрнута на отдельном сервере.
- **Приложение ЕХД** запущено на локальной машине, доступно на порту `3010`.

### Конфигурации тестов

Тестирование проводилось по трём профилям нагрузки:

1. **Single User Load (1 виртуальный пользователь)**  
   Сценарий с последовательным выполнением запросов одним потоком. Предназначен для оценки базовой производительности системы. В рамках итераций выполнялся запрос к целевому эндпоинту с прогрессивно увеличивающимся параметром смещения (`skip`).

2. **Constant Load (5 виртуальных пользователей)**  
   Постоянная фоновая нагрузка в пять параллельных пользователей. Сценарий позволяет оценить поведение приложения в условиях незначительной, но непрерывной параллельной обработки запросов.

3. **Step Load (до 10 виртуальных пользователей)**  
   Ступенчатый профиль с пиковой нагрузкой. Конфигурация предусматривает динамическое изменение интенсивности:
   - Нарастание нагрузки с 0 до 10 виртуальных пользователей.
   - Удержание постоянной нагрузки (10 виртуальных пользователей).
   - Плавное снижение нагрузки до 0.

### Структура проекта

├── test/                  # Сценарии нагрузочных тестов k6
├── results/
│   ├── before_optimize/  # Результаты тестов до оптимизации
│   └── after_optimize/   # Результаты тестов после оптимизации

### Запуск тестов

Для выполнения тестов убедитесь, что приложение ЕХД запущено локально на порту `3010`, а затем выполните команду:

```bash
k6 run test/<имя_сценария>.js
